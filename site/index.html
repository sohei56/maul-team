The problem

Vibe coding decays.
Spec-driven suffocates.

Most real projects live between the two extremes. Maul Team brings Scrum's inspect-and-adapt loop to Claude Code — structured iteration without a complete spec on day one.

✕ Vibe coding

Fast, then fragile

Prompt, accept, repeat. Speed is real — but as the project runs longer, order erodes. No requirements, no tests, no memory of why anything is the way it is.

✕ Spec-Driven Development

Ordered, but upfront-everything

SDD keeps order — but demands you define the whole system before writing a line. Real projects rarely know that much on day one.

✓ The Scrum loop

Inspect and adapt, every Sprint

An AI team plans a Sprint, builds a working Increment, demos it to you, and adapts from your feedback. Requirements sharpen as the product grows — and you never leave the Product Owner seat.

The lifecycle

Real Scrum. Real gates.
No hand-waving.

Every Sprint runs the full ceremony loop — with deterministic termination gates, measured coverage, and cross-model review. Not a prompt chain wearing a Scrum costume.

01

Requirements DefinitionSprint 0

A Requirements Analyst interviews you, researches similar products on the web, and writes requirements.md.

02

Backlog Refinement

Coarse ideas become sprint-ready PBIs with acceptance criteria and scoped paths.

03

Sprint Planning

You approve the Sprint Goal as Product Owner; PBIs are assigned to Developer agents.

04

PBI Pipelineparallel

Each Developer conducts rounds of design → implementation + black-box unit tests → cross-model Codex review, in its own git worktree, with deterministic termination gates and real C0/C1 coverage.

05

Per-PBI Merge

--no-ff merge plus a regression gate; three strikes and the PBI escalates instead of limping forward.

06

Cross-Review

Five aspect-specialized reviewer agents — requirements, functionality, security, maintainability, docs — run in parallel over the Increment.

07

Sprint Review

The team demos the working Increment to you. You inspect the product, not a diff.

08

Retrospective

The team inspects its own process and carries improvements into the next Sprint.

09

Integration Testsrelease train

Design-driven boundary and branch cases, smoke tests, and API/UI automation.

10

UAT & Release

Story-driven UAT walkthrough, defect routing, and the go/no-go release decision.

Or skip the GUI

One command. Same team.

The Mac app wraps the open-source maul-team framework — launch it from any terminal with scrum-start.sh and get the identical pipeline in tmux. Everything runs on your machine, on your Claude subscription — self-hosted by construction.

Agents report facts, not vibes: status transitions, coverage numbers, merge results. When something is genuinely unclear, the team escalates to you instead of guessing.

Your seat

That's the whole job.

Every decision that shapes the product stays with you. Everything else — elicitation, planning, building, reviewing, testing — is the team's problem.

Under the hood

Built like infrastructure,
not a demo.

Hooks enforce the process. State survives restarts. Reviews cross model families. The framework assumes agents will be confidently wrong — and catches it.

Multi-agent orchestration, native to Claude Code

The Scrum Master runs in Delegate mode — it coordinates, never writes code — orchestrating up to 6 parallel Developer agents, one per PBI. Each Developer is itself a conductor, spawning per-round specialist sub-agents.

⌘

Native Mac app

Your AI dev team in one Mac window — editor, Scrum Master terminal, Work Log, and dashboard. SwiftUI, not Electron.

Get started

Early access. Build it yourself
in a couple of minutes.

MaulTeam.app is free, source-available, and pre-notarization, built on the MIT-licensed open-source framework. Today the working path is a source build — four commands, no surprises.

● Available now

Build from source

Clone, build, launch. make-app.sh release produces a universal2 build for Apple Silicon and Intel.

$ git clone git@github.com:sohei56/maul-team.git $ cd maul-team $ sh macapp/scripts/make-app.sh $ open macapp/build/MaulTeam.app

First build fetches SwiftTerm from the network. Prefer the terminal? Skip the app and run sh scrum-start.sh in your project.

Requirements

  • macOS 13+ and Xcode 15+ (Swift 5.9+) for the source build
  • Claude Code CLI ≥ 2.1.172 on PATH — sub-agents spawning sub-agents was unlocked there
  • Python 3.9+ — the launcher validates it

Questions

The parts people
ask about twice.

Short answers to what a "team of AI agents running Scrum" actually means in practice.

?

How do AI agents actually run a Scrum sprint?

A Scrum Master agent plans the Sprint and spawns Developer agents, one per backlog item. Each Developer designs, implements, and writes black-box unit tests in its own git worktree, with a different model family reviewing every artifact. Merges are gated by regression tests; the Sprint ends with a demo to you. It's the ceremony loop, not a prompt chain.

?

Is this autonomous AI development, or am I babysitting prompts?

Sprint execution is hands-off; you make Product-Owner decisions at the gates. --autonomous mode runs fully unattended.

?

Is MaulTeam.app free and self-hosted?

Free and source-available, built on the MIT-licensed maul-team framework, and it runs entirely on your Mac against your own Claude Code install. No SaaS; no code leaves your machine except to the Claude API you already use.

?

Do I need the Mac app, or can I use the terminal?

One shell command runs the identical pipeline in tmux. The Mac app is the friendly front door, not a requirement.

?

How is this different from Devin or other AI software engineers?

A whole team of agents with real Scrum gates, not a single autonomous engineer; source-available and self-hosted, not closed SaaS; and you hold the Product Owner seat.
